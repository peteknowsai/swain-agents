# Boat Art

Every captain gets a new piece of art featuring their boat, every single day. This is a signature Swain feature. Art is print-ready — optimized for posters, stickers, coffee mugs, and koozies.

## Two Ways to Create Boat Art

### 1. In Briefings (automatic)
Include a `boat_art` item in your briefing. The backend auto-creates a boatArt record during `briefing assemble` — every piece of art is saved and gets a shareable URL.

The assemble response includes `artIds`:
```json
{ "success": true, "briefingId": "...", "itemCount": 5, "date": "2026-02-22", "artIds": ["art_a1b2c3d4"] }
```

### 2. On Demand (`swain boat-art create`)
When a captain asks for boat art outside of a briefing — "hey can you make me a pop art version?" — use:
```bash
swain boat-art create --user={{userId}} --style=pop-art --json
```
This generates the image AND saves it as a boatArt record in one shot. Returns:
```json
{ "artId": "art_xxx", "image": "https://...", "styleName": "Pop Art", "boatName": "Fat Cat", "shareUrl": "https://www.heyswain.com/art/art_xxx" }
```

## Shareable URLs

Every piece of boat art — whether from briefings or on-demand — gets a shareable URL:
```
https://www.heyswain.com/art/{artId}
```
Share these with captains. They can send them to friends, post on social media, etc.

To see all art for a captain:
```bash
swain boat-art list --user={{userId}} --json
```

## How It Works

**If the captain has a boat photo on file** — the art is generated by restyling their actual boat photo (image-to-image). The boat stays recognizable but gets transformed into the chosen art style. People, pets, flags, and details are preserved.

**If no photo yet** — the art is generated from a text description based on their boat name, type, and make/model. Still looks great, but a real photo makes it dramatically better.

## Available Styles (30)

**Clean / Illustration:**
- `clean-line` — Precise ink outlines, flat color fills, vector-like
- `colored-pencil` — Warm pencil strokes on white paper
- `blueprint` — White lines on deep blue, engineering schematic look

**Painterly:**
- `watercolor` — Transparent washes, wet-on-wet bleeds
- `japanese-woodblock` — Ukiyo-e style, bold outlines, Hokusai-inspired

**Bold / Graphic:**
- `pop-art` — Ben-Day dots, Lichtenstein-inspired, punchy
- `comic-book` — Bold ink outlines, halftone shading, cel-shaded
- `retro-poster` — 1950s travel poster, flat shapes, limited palette
- `neon-glow` — Glowing neon outlines on dark background, synthwave
- `stencil` — Banksy-inspired, high contrast spray paint aesthetic

**Screen Print / Production:**
- `screen-print` — 2-3 color separation, vintage tee aesthetic
- `risograph` — Overlapping color layers, grainy indie poster look

**Nautical / Maritime:**
- `nautical-tattoo` — Traditional sailor tattoo, bold old-school lines
- `nautical-engraving` — Copper plate etching, fine crosshatch detail
- `vintage-chart` — Aged nautical chart, sepia parchment aesthetic

**Coastal / Lifestyle:**
- `tiki` — Tiki bar poster, tropical retro polynesian
- `vintage-florida` — 1960s Florida postcard, sun-bleached pastels
- `surf-art` — 1970s surf culture, earthy flat color palette

**Craft / Handmade:**
- `embroidery` — Embroidered patch look, visible thread texture
- `paper-cutout` — Layered paper cut art, dimensional depth

**Modern / Digital:**
- `flat-vector` — Clean minimal vector, no gradients, app-icon crisp
- `cartoon` — Bright and playful, exaggerated fun proportions
- `pixel-art` — 16-bit retro game aesthetic, visible square pixels

**Monochrome:**
- `mono-navy` — Single navy blue on white, timeless maritime
- `chalk` — White chalk on dark chalkboard, hand-drawn texture
- `sepia-sketch` — Warm brown tones on cream, old master drawing feel

**Premium / Specialty:**
- `art-deco` — Gold and navy, 1920s Gatsby glamour
- `linocut` — Bold carved lines, black and white with one accent color
- `gold-foil` — Metallic gold line art on dark navy, luxury nautical
- `psychedelic` — 1960s concert poster, swirling vibrant colors

## Commands

### Generate + save (on-demand use)
```bash
swain boat-art create --user={{userId}} --json                    # Random style
swain boat-art create --user={{userId}} --best --json             # Best style for boat type
swain boat-art create --user={{userId}} --style=pop-art --json    # Specific style
```

### List all art for a captain
```bash
swain boat-art list --user={{userId}} --json
```

### Generate image only (for briefings)
```bash
swain card boat-art --user={{userId}} --best --json
swain card boat-art --user={{userId}} --json
swain card boat-art --user={{userId}} --style=nautical-tattoo --json
```
Use the `image`, `styleName`, and `boatName` from the result to build a `boat_art` briefing item. The backend creates the record during assembly.

## First Briefing — Best Style

The captain's first piece of boat art should be a wow moment. Use `--best` to automatically pick the style that'll look most impressive for their specific boat:

1. Run `swain card boat-art --user={{userId}} --best --json`
2. This picks the ideal style based on boat type/make (sportfishers get clean-line, sailboats get watercolor, bay boats get vintage-florida, etc.)
3. Take the `image`, `styleName`, and `boatName` from the result and include as a `boat_art` item in the briefing
4. Add commentary like: "One of my favorite things I do for you — every day, I create a new piece of art featuring [boat name]. Here's today's. Eventually you'll be able to print these on stuff too."
5. One image, maximum impact. Tomorrow they get a different style.

## Daily Briefings — One Art Card Per Day

Every daily briefing should include **one boat art image**:

1. Run `swain card boat-art --user={{userId}} --json` (random style)
2. Include the result as a `boat_art` item in the briefing with a brief commentary
3. Rotate through styles — try not to repeat the same style two days in a row

## On-Demand Requests

When a captain asks for boat art in conversation:

1. Run `swain boat-art create --user={{userId}} --style=<requested-style> --json`
2. Send them the `shareUrl` via WhatsApp
3. The art is automatically saved and shows up in their art collection in the app

## Getting a Boat Photo

A real photo makes the art dramatically better. Ways to get one:

- **Ask during onboarding**: "Got a pic of [boat name]? I use it to make custom art for you every day."
- **Ask in the first briefing**: Include a text item asking for a photo
- **They may send it via WhatsApp** — when they do, save it:
  ```bash
  swain user upload-boat-image {{userId}} --url=<photo_url>
  ```
  or if you have the file locally:
  ```bash
  swain user upload-boat-image {{userId}} --file=<path>
  ```

Once a photo is on file, all future boat art automatically uses it for image-to-image generation.

## Art Display Mode (iOS)

`boat_art` briefing items trigger **art display mode** on iOS: full-bleed image rendering on a white background, boat-centered layout, automatic bookmarking, and the style name as title. The backend creates the record with `displayMode: "art"` during briefing assembly — you don't set this manually.

## Architecture Decision: Two Sessions Per Advisor
- **System session** (`agent:advisor:main`) — Commodore coordination, heartbeats, triggers. Advisor uses `message` tool to reach captain.
- **Captain session** (`agent:advisor:linq:dm:+phone`) — 1:1 iMessage. Auto-delivery. Every word = iMessage.
- This matches Bobby's existing pattern (main + slack:channel)

## First Contact Flow (Final)
1. System triggers advisor via `openclaw agent --agent <id> --message "New captain assigned..."`
2. Advisor pulls profile from `skip user get`
3. Advisor sends intro via `message` tool → Linq → iMessage
4. Captain replies → creates linq-bound captain session
5. Conversation continues in captain session
